# Interview Questions and Resources

Welcome to the collection of IT interview questions, programming facts, and useful resources. 

The main purpose of this repository is to serve as a knowledge base for:
- 🧠 **Interview Preparation** - A categorized collection of common questions, concepts, and technical problems encountered during software engineering interviews.
- 💡 **Programming Trivia** - Interesting facts, neat tricks, and deep-dives into how specific technologies work under the hood.
- 🔗 **Useful Resources** - Curated links to valuable documentation, GitHub repositories, and technical articles.

## Repository Structure

The questions are organized by topic into separate Markdown files within the `questions/` directory:

*   [C# & Object-Oriented Programming](questions/csharp.md) - OOP concepts, Value/Reference types, Delegates, Threading, LINQ, Memory Management.
*   [.NET & ASP.NET Core](questions/dotnet.md) - Framework concepts, Dependency Injection, Request Pipeline, EF Core.
*   [Architecture & Design Patterns](questions/architecture_and_design.md) - SOLID principles, Microservices, Testing, TDD, DDD.
*   [SQL & Databases](questions/sql.md) - Triggers, Views, Indexes, Temporary tables.
*   [Frontend & Web Technologies](questions/frontend.md) - JavaScript concepts (Event loop, Promises), TypeScript semantics.
*   [General & HR](questions/general_and_hr.md) - Soft skills, past experience, code review practices.

Additionally, you can find a curated list of interesting links here:
*   [Resources and Links](resources_and_links.md)
*   [External Projects](projects.md)

## Contributing / AI Assistance Guidelines

If you are using an AI assistant to help maintain this repository, please note the guidelines outlined in the [System Prompt](system_prompt.md). All new questions and answers should ideally be clear, technically accurate, and formatted consistently using Markdown.
